# 1. Why Flutter

```
하나의 언어와 프레임워크로 모든 것을 만들 수 있다.
(IOS, 안드로이드, 웹, 맥OS, 윈도우, 리눅스, 임베디드)
```

# 2. How Flutter Works

```
Flutter는 게임 엔진처럼 동작
Flutter는 무언가를 렌더링할 때 운영체제와 얘기하는 것이 아니라
Flutter 엔진이 캔버스에 무언가를 그린다.
따라서 화면 상의 모든 픽셀들을 자유자재로 다룰 수 있다.

Framework(dart)
  |
Engine(C/C++) : 어플리케이션의 실제 UI를 렌더링 해줌
  |
Embedder(Platform-specific) : - 엔진을 가동시키는 "runner" 프로젝트
                              - iOS, 안드로이드, 맥OS, 윈도우, 리눅스를 위한 다양한 Embedder가 존재한다.
                              - 호스트 플랫폼 상에서 엔진을 가동시키는 역할을 한다.


유저가 애플리케이션을 실행시키면
Flutter 엔진을 동작시키는 "runner" 프로젝트가 실행되고
엔진이 Dart 코드(SDK와 내가 짠 코드)를 실행시킴

다른 프레임워크들은 운영체제와 직접 소통하는 방식을 취한다.
하지만 Flutter는 단순히 엔진을 어플리케이션 내부에 집어넣고 우리가 짠 Dart 코드를 컴파일 한다.
유저가 맥O, 윈도우, iOS, 안드로이드 어플리케이션을 실행시킬 때
엔진을 가동시키는 "runner" 프로젝트를 실행시키고
그 이후에 엔진이 모든 UI를 프레임워크와 함께 그려준다.
```

# 3. Flutter vs React Native

```
React Native :
네이티브 앱 운영체제 상에서 가능한 위젯을 사용하고 싶을 때
호스트 운영체제가 제공해주는 스타일에 의존하고 싶을 때

Flutter :
커스터마이징 된 디자인을 만들고 싶을 때
iOS 혹은 안드로이드 앱처럼 안보이게 만들고 싶을 때
```
